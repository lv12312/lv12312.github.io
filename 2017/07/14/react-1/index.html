1. 容易迷路的this

建议使用 ES6 的箭头函数（俗称胖箭头）=> 避免 this 的迷失。不熟悉的同学可参考文档，https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Functions/Arrow_functions

箭头函数最大的特点就是不会绑定函数自己的this，所以在函数里面用的this都是来自函数所属上下文的this。

3. 生命周期

组件的生命周期包含多个生命周期方法，带有will 的方法发生在某些事件发生之前，带有did的方法发生在某些事件发生之后。

3.1 组装

  • constructor() ，初始化默认的state
  • componentWillMount()，通常用来作为服务器端的页面初始化
  • render()
  • componentDidMount，在页面装载之后调用，通常用来服务器渲染页面的辅助信息，如一些复选框的选项等。

3.2 更新

  • componentWillReceiveProps()，组装好的组件如果收到新的 props 该方法将会被调用，可用来处理来自服务端异步返回的数据。
  • shouldComponentUpdate()，使用该方法让 React 知道组件的输出有没有被目前的 state 或者 props 影响到，很少使用到。默认返回 true, 如果返回 false, 后面的3个方法就不会被调用。
  • componentWillUpdate()，通常用来做渲染前的数据处理，但是不能调用this.setState(), 如果有需要应该放在componentWillReceiveProps()
  • render()，不要在该方法里面调用this.setState()
  • componentDidUpdate()，更新结束之后调用的方法

3.2 卸载

组件从 DOM 中移除时调用的方法，对本页面进行清理，常被用来清除timer。

  • componentWillUnmount()
