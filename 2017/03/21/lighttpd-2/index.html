简介

Lighttpd 使用多进程的网络服务模型，一个监控进程 (Wachter)，多个工作进程 (Workers)。监控进程负责监视工作进程，一旦有工作进程出错退出，会重新创建新工作进程；工作进程负责接收客户端请求并提供服务。默认情况下Lighttpd的配置参数 server.max-worker 为 0 ，此时单工作进程运行，没有监控进程。一般情况下该参数设置为服务器核心数。
